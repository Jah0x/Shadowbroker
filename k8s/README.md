# Kubernetes deployment

Manifest file: `k8s/shadowbroker.yaml`.

## 1) (Optional) Authenticate to GHCR for private image pulls

If the repository/package is private, create an image pull secret:

```bash
kubectl create secret docker-registry ghcr-creds \
  --namespace shadowbroker \
  --docker-server=ghcr.io \
  --docker-username=<github-username> \
  --docker-password=<github-personal-access-token>
```

Then add `imagePullSecrets` to both Deployments if needed.

## 2) Configure API keys

Edit `stringData` in the `shadowbroker-secrets` Secret or patch it with kubectl before applying.

## 3) Deploy

```bash
kubectl apply -f k8s/shadowbroker.yaml
kubectl -n shadowbroker get pods,svc
```

Frontend service is exposed on port `80` (`LoadBalancer`).

## 4) Pin a specific release tag (recommended)

Instead of `:latest`, update images to a git tag or SHA generated by the publish workflow:

- `ghcr.io/<owner>/<repo>-frontend:<tag>`
- `ghcr.io/<owner>/<repo>-backend:<tag>`

Example:

```bash
kubectl -n shadowbroker set image deploy/frontend frontend=ghcr.io/bigbodycobain/shadowbroker-frontend:main
kubectl -n shadowbroker set image deploy/backend backend=ghcr.io/bigbodycobain/shadowbroker-backend:main
```
